◆ trie_findall()

TrieIterator* trie_findall ( Trie *  trie,
const char *  key_prefix,
size_t  max_len 
)

Create an iterator to cover all keys with a given prefix and maximum size.

NULL will be returned either if the amount of free memory required is not available or if there are no keys to begin with subject to the given prefix and length constraints.

Keys will be enumerated in ascending lexicographic order, i.e. strcmp(key1, key2) will be negative if key1 is enumerated before key2.

Enumeration without a maximum length constraint can be done by passing the return value of trie_maxkeylen_added in the third argument.

Parameters
trieTrie context
key_prefixC-string prefixing all keys to enumerate
max_lenUpper bound on the lengths of the keys to enumerate
Returns
Valid iterator or NULL

char *  key,

Insert a key-value pair into a trie.

Values inserted with a pre-existing key will replace the corresponding pre-existing value and the pre-existing value will be destroyed.

The insertion will fail either if the required amount of free memory is not available or if val is NULL. In case of failure, the trie is left unchanged.

◆ trie_iter_next()

void trie_iter_next ( TrieIterator **  iter_p)

Advance an iterator to the next valid (key, value) pair.

Iterators are invalidated either if the iterator has ended or if the amount of free memory required to proceed with the iteration is not available. *iter_p, if invalidated, is set to NULL.

Behavior of a valid iterator, after any trie modification, is undefined.

Parameters
iter_pPointer to valid iterator or NULL
